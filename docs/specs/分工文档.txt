## 公共接口定义（必须严格遵守）

三人分别开发，通过以下接口协作。**接口由 Person C 在 Day 0 定义并提交到 `src/interfaces.py`**。

### 接口 1: AIEngine（Person A 实现，Person C 调用）

```python
# src/interfaces.py
from dataclasses import dataclass, field
from typing import List, Dict, Optional

@dataclass
class MatchResult:
    """AI 匹配结果，由 LLM 返回"""
    component_id: Optional[int]       # 匹配到的器件 ID（无匹配则为 None）
    component_name: str               # 器件名称
    cabinet_id: Optional[int]         # 所在柜门编号
    confidence: float                 # 匹配置信度 0.0-1.0
    response_text: str                # LLM 生成的自然语言回复（播报用）
    alternatives: List[Dict] = field(default_factory=list)  # 备选器件列表

class AIEngine:
    """AI 语义引擎接口"""

    def initialize(self) -> bool:
        """加载 rkllm 模型，返回是否成功"""
        ...

    def process_query(self, query_text: str, context: Dict = None) -> MatchResult:
        """
        处理用户自然语言查询，返回匹配结果。
        Person C 将语音识别的文本传入，得到结构化的 MatchResult。

        Args:
            query_text: 用户原始语音文本，如 "我需要一个5V的稳压芯片"
            context: 可选上下文（当前用户、历史对话等）

        Returns:
            MatchResult: 包含匹配器件和自然语言回复
        """
        ...

    def shutdown(self):
        """释放模型资源"""
        ...
```

@dataclass
class AuthResult:

class HardwareController:
    """硬件控制接口"""

    def initialize(self) -> bool:
        """初始化所有硬件：舵机、GPIO、NFC、摄像头"""
        ...

    # ── 柜门控制 ──

        """打开指定柜门（舵机90°），返回成功与否"""

        ...

    # ── 身份验证 ──
    def read_nfc_card(self, timeout: float = 10.0) -> Optional[str]:

**核心代码**：
```python
# src/ai/rkllm_engine.py
from rkllm import RKLLM

class RKLLMEngine:
    def __init__(self, model_path: str):
        self.llm = RKLLM(model_path)
        self.llm.init()

    def chat(self, prompt: str, max_tokens: int = 256) -> str:
        """调用 LLM 推理，返回生成的文本"""
        return self.llm.generate(prompt, max_tokens=max_tokens)
```

#### A2. 器件匹配 Prompt 工程（1天）
```
# src/ai/prompt_templates.py

SYSTEM_PROMPT = """你是一个实验室器件管理助手。用户会用自然语言描述需要的器件，
你需要从数据库中找出最匹配的器件。

请严格按以下JSON格式输出，不要输出其他内容：
{
    "component_id": 数字或null,
    "component_name": "器件名称",
    "cabinet_id": 数字或null,
    "confidence": 0.0到1.0的浮点数,
    "response": "给用户的自然语言回复",
    "alternatives": [{"id": 1, "name": "备选1"}, ...]
}
"""

def build_query_prompt(user_query: str, component_catalog: str) -> str:
    """构建查询 prompt"""
    return f"""{SYSTEM_PROMPT}

当前器件数据库：
{component_catalog}

用户说：「{user_query}」

请匹配最合适的器件："""
```

## 独立开发与测试策略

每个 Person 都可以不依赖其他人独立开发：

| Person | 独立测试方式 |
|--------|-------------|
| **A** | 用模拟的器件目录文本 + CLI 交互，验证 LLM 匹配准确率 |
| **B** | 不用数据库，直接命令行调用舵机/传感器/读卡器接口 |
| **C** | 用 Mock AIEngine + Mock HardwareController 先跑通 GUI 和数据库 |

**C 需要提前准备 Mock 实现**：
```python
# src/mocks.py (开发用)
class MockAIEngine(AIEngine):
    def process_query(self, query_text, context=None):
        return MatchResult(
            component_id=1, component_name="LM7805",
            cabinet_id=1, confidence=0.95,
            response_text="为您找到LM7805，5V稳压芯片，在1号柜"
        )

class MockHardwareController(HardwareController):
    def __init__(self):
        self.doors = {i: "closed" for i in range(1, 9)}
    def open_locker(self, lid):
        self.doors[lid] = "open"; return True
    def close_locker(self, lid):
        self.doors[lid] = "closed"; return True
```
