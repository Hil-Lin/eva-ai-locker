# 智能储物柜系统设计规范

## 项目概述

### 项目背景
针对实验室器件繁多、选型困难的痛点，基于瑞芯微RK3588平台设计一款本地部署AI的嵌入式器件智能储物柜。用户通过自然语言描述功能需求，系统利用本地轻量化AI模型进行语义匹配，自动推荐最合适的器件，并联动控制对应柜门开锁及借还记录。

### 设计目标
1. **本地AI推理**：全程无需联网，响应快速，保障数据隐私
2. **自然语言交互**：用户通过语音描述需求，系统语音回复并推荐器件
3. **双重身份验证**：支持人脸识别和校园卡NFC识别，两级权限控制
4. **智能器件管理**：AI语义匹配推荐最合适的实验室器件
5. **模块化设计**：融合边缘AI推理与嵌入式控制，便于扩展和维护

### 技术指标
- **主控平台**：ELF 2开发板（RK3588芯片）
- **AI算力**：6TOPS NPU，支持本地推理
- **响应时间**：语音识别+AI匹配+开锁 < 5秒
- **存储容量**：支持100+种器件信息，1000+条借还记录
- **柜门数量**：8个柜门（PWR.A153 B 8路舵机通道），可级联扩展
- **工作环境**：全程无网，本地运行

## 工作流程设计

### 用户取物流程（5个步骤）

#### 步骤1：语音提出需求
- **输入方式**：语音输入自然语言描述
- **示例**："我需要一个5V的稳压芯片"
- **技术要求**：Vosk离线语音识别库

#### 步骤2：身份验证
- **普通学生**：人脸识别 **或** 校园卡NFC
- **管理员**：人脸识别 **+** 管理员密码
- **验证失败处理**：提示重试或使用备用方式

#### 步骤3：AI语义匹配
1. **语音转文本**：自然语言描述转为文本
2. **关键词提取**：提取电压、类型、参数等关键词
3. **规则匹配**：基于器件特征数据库进行匹配
4. **相似度计算**：加权评分算法，返回排名前3的推荐

#### 步骤4：柜门控制
- **自动开锁**：系统控制对应柜门锁打开
- **语音提示**："柜门#03已打开，请取用器件"
- **状态显示**：触摸屏显示柜门状态

#### 步骤5：借还记录
- **自动记录**：用户、器件、柜门、时间信息
- **本地存储**：SQLite数据库存储
- **历史查询**：管理员可查询借还记录

### 异常处理流程
- **身份验证失败**：提示重新验证，支持备用验证方式
- **器件未找到**：AI推荐相似器件，显示库存状态
- **柜门故障**：系统报警，尝试备用柜门
- **电源异常**：自动保存数据，安全关机

## 硬件系统设计

### 硬件模块清单

#### 1. 主控模块
- **ELF 2开发板 (RK3588)**
  - CPU: 四核Cortex-A76 + 四核Cortex-A55
  - NPU: 6TOPS算力，支持本地AI推理
  - 内存: 8GB LPDDR4
  - 存储: 64GB eMMC
  - 接口: USB 3.0×2, USB 2.0×2, HDMI, GPIO, I2C, SPI

#### 2. 舵机控制模块
- **小R科技 PWR.A153 B 扩展板**
  - 8路标准舵机接口（黑-GND / 红-5V / 黄-PWM）
  - 板载 74HC244 缓冲芯片，隔离 GPIO 保护 RK3588
  - 板载 DC 电源接口 + 5V 稳压电路，舵机独立供电
  - 直接插入 RK3588 40pin GPIO 排针（树莓派兼容）
  - 通过 RK3588 GPIO 硬件 PWM 控制舵机角度
  - 预留全部 GPIO 引脚，可扩展传感器
  - 控制8个 SG90 舵机，驱动柜门锁扣机构

#### 3. 传感器模块
- **USB摄像头模块**
  - 200万像素，支持人脸采集和识别
  - 自动白平衡，低光照补偿
- **NFC读卡器**
  - 支持ISO14443A/B协议
  - 读取校园卡UID
  - UART接口连接

#### 4. 执行器模块
- **柜门舵机**
  - SG90舵机，5V供电，扭矩1.5kg·cm
  - 每个柜门独立舵机控制锁扣
- **门状态传感器**
  - 微动开关/光电开关检测门开关
  - 防止误关门或未关门

#### 5. 人机交互模块
- **7寸触摸显示屏**
  - 1024×600分辨率，电容触摸
  - HDMI接口连接
- **麦克风阵列**
  - 双麦克风，语音降噪
- **扬声器模块**
  - 3W功率，语音播报

#### 6. 电源管理模块
- **输入**: 12V/5A直流电源
- **输出分配**:
  - 5V → RK3588开发板 (USB供电)
  - 5V → PWR.A153 B DC插座 (扩展板+舵机独立供电，推荐5V/3A以上)
  - 3.3V → 传感器
- **供电隔离**: RK3588和舵机分开供电，避免舵机堵转拉低主控电压
- **保护功能**: 过流保护、短路保护

### 硬件接口设计

| 模块 | 接口类型 | 连接到 | 功能描述 |
|------|----------|--------|----------|
| RK3588主控 | USB 2.0 | 摄像头模块 | 人脸图像采集 |
| RK3588主控 | UART | NFC读卡器 | 校园卡信息读取 |
| RK3588主控 | GPIO PWM (×8) | PWR.A153 B 扩展板 | 8路舵机PWM控制 (通过74HC244缓冲) |
| RK3588主控 | GPIO × 12 | 门状态传感器 | 检测柜门开关状态 |
| RK3588主控 | HDMI | 触摸显示屏 | 图形界面显示 |

> **注意**：PWR.A153 B 通过 40pin 排针直接插入 RK3588，扩展板预留全部 GPIO 引脚用于传感器等扩展。

### 实物设计考虑
- **机箱设计**: 钣金外壳，尺寸约400×300×200mm
- **散热设计**: RK3588主动散热（小型风扇），机箱通风孔
- **安装维护**: 模块化安装，线缆管理，预留扩展接口

### AI语义匹配实现方案

#### 器件特征数据库
```json
{
  "components": [
    {
      "id": 1,
      "name": "LM7805",
      "category": "稳压芯片",
      "voltage": 5.0,
      "current": 1.0,
      "package": "TO-220",
      "cabinet_id": 3,
      "keywords": ["稳压", "5V", "线性稳压", "电源芯片"],
      "stock": 5
    }
  ]
}
```

#### 规则匹配引擎
1. **语音转文本**: "我需要一个5V的稳压芯片"
2. **中文分词**: ["我", "需要", "一个", "5V", "的", "稳压", "芯片"]
3. **关键词提取**: ["5V", "稳压", "芯片"]
4. **规则匹配**:
   - 电压匹配 (5V)
   - 类型匹配 (稳压芯片)
   - 参数匹配 (电流、封装等)

#### 相似度计算算法
```
评分 = 权重1 × 电压匹配度 +
       权重2 × 类型匹配度 +
       权重3 × 关键词匹配数

返回: 排名前3的推荐器件
```

### 数据存储设计

#### SQLite数据库表设计
```sql
-- 器件信息表
CREATE TABLE components (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT,
    voltage REAL,
    current REAL,
    cabinet_id INTEGER,
    keywords TEXT,
    stock INTEGER
);

-- 用户信息表
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    card_id TEXT UNIQUE,
    face_feature BLOB,
    permission_level INTEGER  -- 1:普通学生, 2:管理员
);

-- 借还记录表
CREATE TABLE records (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,
    component_id INTEGER,
    cabinet_id INTEGER,
    borrow_time TIMESTAMP,
    return_time TIMESTAMP
);
```

#### 数据流设计
- **启动时加载**: 器件数据库、用户信息预加载到内存
- **实时更新**: 借还记录实时写入数据库
- **定期备份**: 重要数据定期备份到SD卡
- **容错机制**: 数据库损坏时从备份恢复

## 安全与权限设计

### 两级权限体系
1. **普通学生 (权限等级1)**
   - 验证方式: 人脸识别 **或** 校园卡NFC
   - 功能权限: 查询器件、借用器件、查看个人记录

2. **管理员 (权限等级2)**
   - 验证方式: 人脸识别 **+** 管理员密码
   - 功能权限: 所有学生功能 + 系统管理、用户管理、数据查询

### 安全机制
- **数据加密**: 敏感数据本地加密存储
- **操作日志**: 所有关键操作记录日志
- **防拆机检测**: 机箱开启检测与报警
- **异常锁定**: 多次验证失败后锁定账户
