# 智能储物柜用户取物流程设计规范

## 设计背景

本设计文档基于现有《2026-04-16-智能储物柜设计.md》规范进行扩展，重点定义用户取物流程、界面交互设计和管理员功能模块。针对嵌入式芯片与系统设计竞赛要求，完善系统的用户交互体验和管理功能。

### 设计目标

1. **完善用户取物流程**：定义清晰的借、还、设置三大功能模块
2. **优化界面交互**：设计直观的触摸屏界面，支持语音+触摸混合交互
3. **扩展管理员功能**：实现完整的管理员控制面板
4. **集成条形码模块**：新增条形码扫描功能，支持归还和数据库更新
5. **保持架构一致性**：在现有四层架构基础上模块化扩展

### 设计原则

- **用户友好**：界面简洁直观，操作流程清晰
- **安全可靠**：双重身份验证，权限分级控制
- **模块化设计**：新功能模块化集成，保持架构清晰
- **竞赛适应性**：符合嵌入式系统设计竞赛评审要求

### 新增硬件模块：条形码扫描器

- **接口类型**：USB HID（即插即用）
- **扫描类型**：支持Code 128, Code 39, EAN-13, UPC-A, UPC-E等常见格式
- **连接方式**：直接连接到RK3588的USB 2.0接口
- **工作模式**：键盘模拟模式，扫描数据直接输入为文本
- **扫描速度**：≥100次/秒
- **解码能力**：支持倾斜、扭曲、破损条形码识别
- **照明方式**：LED照明，适应不同光照条件
- **安装位置**：固定在柜体侧面，倾斜角度15°，便于用户扫描
- **技术规格**：
  - 电源：5V DC，USB供电
  - 工作电流：<100mA
  - 扫描距离：5-30cm
  - 扫描角度：±40°
  - 环境温度：0-50°C
  - 防护等级：IP54（防尘防溅）

### 硬件连接关系

```

      ↓
USB条形码扫描器
```

### 借用流程详细设计

#### 步骤1：模式选择

用户进入借用流程后，首先选择借用方式：

- **AI辅助借用**：通过语音描述需求，AI推荐最合适的器件
- **直接借用**：手动选择器件或柜子进行借用

#### 步骤2：身份验证

采用并行验证方式，用户任选其一：

- **人脸识别**：面向摄像头进行识别
- **校园卡NFC**：将校园卡靠近读卡器

**验证失败处理**：

- 提示重新验证
- 支持切换验证方式
- 连续失败3次后短暂锁定

#### AI辅助借用流程（4步）

1. **语音输入需求**：用户通过语音描述所需器件功能
2. **AI语义匹配**：系统推荐排名前3的器件
3. **屏幕确认选择**：在触摸屏上显示推荐结果供选择
4. **确认借用**：AI语音询问"是否借用？"，用户确认后开锁

#### 直接借用流程（4步）

1. **选择方式**：三种选择方式：
   - 按借用历史：显示最近借用记录
   - 按器件分类：浏览器件分类树
   - 按柜子编号：直接输入或选择柜子编号
2. **选择柜子**：手动选择目标柜子
3. **绑定借用信息**：记录用户-器件-柜子关系
4. **锁定柜子**：标记"锁01"，对应柜子状态灯显示借出状态

### 归还流程设计（4步）

1. **扫描条形码**：使用条形码扫描器扫描器件上的条形码
2. **系统识别**：自动识别对应柜子
3. **打开柜门**：自动打开对应柜门
4. **更新记录**：更新借还记录状态，柜子状态灯恢复

## 管理员设置功能设计

### 管理员验证流程

采用双重验证确保系统安全：

1. **人脸识别验证**：先进行管理员人脸识别
2. **管理员密码输入**：通过人脸识别后输入6位数字密码

**安全机制**：

- 连续3次验证失败后锁定管理员账户30分钟
- 密码加密存储：使用SHA-256加盐哈希算法存储密码哈希值
- 操作日志记录所有管理员操作，包括时间、IP地址、操作类型
- 会话管理：管理员会话超时时间15分钟
- 密码策略：6位数字密码，强制90天更换
- 防暴力破解：验证失败延迟递增（1s, 2s, 4s, ...）

#### 1. 数据库更新（条形码扫描绑定）

#### 管理员操作日志表

```sql
CREATE TABLE admin_logs (
    id INTEGER PRIMARY KEY,
    admin_id INTEGER NOT NULL,         -- 管理员ID
    operation_type TEXT NOT NULL,      -- 操作类型
    operation_subtype TEXT,            -- 操作子类型
```

### 关键算法实现

#### 条形码处理流程

1. **硬件输入**：USB HID设备接收扫描数据，通过Linux输入子系统获取
2. **数据解析**：解析键盘输入为文本字符串，处理特殊字符（回车、前缀、后缀）
3. **格式验证**：验证条形码格式有效性，支持多种编码格式
4. **去重处理**：防止同一条形码在短时间内重复扫描（防抖间隔500ms）
5. **数据库查询**：查询条形码映射表，支持模糊匹配和容错处理
6. **业务处理**：根据场景执行相应操作（归还/数据库更新/查询）
7. **反馈提示**：通过声音和界面提示扫描结果（成功/失败）

#### 条形码数据格式处理

- **前缀/后缀处理**：支持配置前缀和后缀字符过滤
- **数据校验**：支持校验位验证（如EAN-13的校验位）
- **格式转换**：统一转换为内部标准格式存储
- **错误处理**：无效格式、无法识别的条形码处理流程
- **日志记录**：记录所有扫描操作，用于审计和故障排查

#### AI语义匹配优化

1. **语音转文本**：Vosk离线语音识别
2. **中文分词**：jieba分词库
3. **关键词提取**：电压、类型、参数等关键词
4. **规则匹配**：基于器件特征数据库进行匹配
5. **相似度计算**：加权评分算法，返回排名前3的推荐

### 通信协议设计

#### RK3588 → PWR.A153 B 舵机控制协议

```
GPIO 硬件 PWM 直连控制:
- RK3588 GPIO PWM → PWR.A153 B 74HC244 缓冲 → 舵机信号线
- PWM频率: 50Hz（舵机标准，周期20ms）
- 0° (脉宽0.5ms) → 锁扣闭合
- 90° (脉宽1.5ms) → 锁扣打开
- 柜门操作流程: 设置占空比→延时800ms→停止PWM（防堵转）

PWR.A153 B 接口（黑-红-黄排针）：
- 黑色(GND) ← 舵机棕色/黑色线
- 红色(5V)  ← 舵机红色线（扩展板5V稳压供电）
- 黄色(PWM) ← 舵机橙色/黄色线（74HC244缓冲的GPIO信号）
```

#### USB条形码扫描器数据格式

```
扫描数据示例：
"COMP-20240423-001"  # 器件条形码
"USER-20240001"      # 用户卡号（如支持）
```
