**三个"AI"都不用你训练，全部是开箱即用的预训练模型。你的工作是写代码把它们串起来。**

| 模块 | 模型 | 来源 | 需要训练？ | 需要做什么？ |

## 1. AI 模型（DeepSeek）怎么跟数据库联系？

### 核心原理：把数据库变成"小抄"塞进 prompt

DeepSeek 模型**不直接访问数据库**。你把数据库内容写成一段文字，塞进 prompt，模型读了之后做匹配。

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  Person C 的 DataManager                              │
│  get_all_components_for_llm()                         │
│       │                                              │
│       ▼                                              │
│  "1. LM7805 | 5V稳压芯片 | 柜门1 | 关键词:稳压,5V     │
│   2. AMS1117-3.3 | 3.3V LDO | 柜门2 | 关键词:3.3V    │
│   3. ESP32-WROOM-32 | WiFi模块 | 柜门5 | ..."         │
│       │                                              │
│       ▼                                              │
│  ┌─────────────────────────────────────┐             │
│  │          Prompt（一整段文字）         │             │
│  │                                      │             │
│  │  你是一个器件管理助手。               │             │
│  │  当前库存：[上面那段器件列表]          │             │
│  │  用户说："我要5V稳压芯片"             │             │
│  │  请用JSON返回匹配结果                 │             │
│  └─────────────────────────────────────┘             │
│       │                                              │
│       ▼                                              │
│  ┌──────────────────────┐                            │
│  │  DeepSeek 1.5B 模型   │                            │
│  │  (RK3588 NPU 推理)    │                            │
│  └──────────────────────┘                            │
│       │                                              │
│       ▼                                              │
│  {"component_id": 1,                                 │
│   "component_name": "LM7805",                        │
│   "cabinet_id": 1,                                   │
│   "confidence": 0.95}                                │
│       │                                              │
│       ▼                                              │
│  Person C 拿到 component_id=1                         │
│  → DataManager.get_by_id(1) → 查完整信息              │
│  → HardwareController.open_locker(1) → 开锁           │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 关键接口

```python
# Person C 提供 → Person A 调用
catalog = data_manager.get_all_components_for_llm()
# 返回: "1. LM7805 | 稳压芯片 | 5V | 柜门1\n2. ..."

# Person A 返回 → Person C 调用
result = ai_engine.process_query("我要5V稳压芯片")
# 返回: MatchResult(component_id=1, cabinet_id=1, confidence=0.95)
```

## 2. 语音识别怎么整合？

### 语音→文本→LLM→文本→语音

```
用户说话                    AI处理                      系统响应
───────                    ──────                      ────────

"我要5V稳压芯片"
    │
    ▼
┌──────────┐    文本                 JSON              语音
│ 麦克风    │    "我要5V             {id:1,             "为您找到
│   ↓       │    稳压芯片"   ──→     name:"LM7805",     LM7805...
│ Vosk模型  │                      cabinet_id:1}       请取用"
│   ↓       │                         │                  ↑
│ 语音→文本 │                         ▼                  │
└──────────┘                   ┌──────────────┐    ┌──────────┐
    Person A                   │ DeepSeek LLM │    │ eSpeak   │
    (语音识别)                  │ (语义理解)    │    │ (语音合成)│
                               └──────────────┘    └──────────┘
                                   Person A           Person A
```

### 数据流（全是字符串传递）

```python
# 1. 语音 → 文本（Vosk）
audio_bytes = microphone.read()
text = vosk_recognizer.recognize(audio_bytes)
# → "我要一个5V的稳压芯片"

# 2. 文本 → 结构化结果（DeepSeek LLM）
result = ai_engine.process_query(text)
# → MatchResult(component_id=1, component_name="LM7805", response_text="为您找到LM7805...")

# 3. 播报回复（eSpeak）
espeak.speak(result.response_text)
# → 扬声器播放："为您找到LM7805，5V稳压芯片，存放在1号柜"
```

**没有复杂的向量、嵌入、特征——全程只用字符串传递。**

## 3. 人脸识别怎么工作？

### 不是"AI 训练"，而是"特征比对"

```
             注册阶段（只需做一次）
             ─────────────────
             用户对着摄像头
                  │
                  ▼
         ┌────────────────┐
         │ OpenCV 拍照      │
         │ Dlib 检测人脸    │
         │ Dlib 提取特征    │
         │ → 128 个浮点数   │
         │   [0.12, -0.34,  │
         │    0.56, ...]    │
         └────────┬───────┘
                  │
                  ▼
         Person C 的 DataManager
         INSERT INTO users (face_features)
         VALUES ('[0.12, -0.34, 0.56, ...]')


             验证阶段（每次借用时）
             ─────────────────
             来人对着摄像头
                  │
                  ▼
         ┌────────────────┐
         │ 同样的流程       │
         │ 提取 128 维特征  │
         │ [0.11, -0.35,   │
         │  0.55, ...]     │
         └────────┬───────┘
                  │
                  ▼
         跟数据库里所有人的特征逐一比对
              │
              ▼
         ┌────────────────────────────┐
         │ 欧氏距离 = sqrt(Σ(a-b)²)    │
         │                            │
         │ 距离 < 0.6 → 同一个人       │
         │ 距离 > 0.6 → 不是同一个人    │
         └────────────────────────────┘
```

### 代码（就这几行）

```python
import numpy as np

# 两个 128 维特征向量
feat_registered = [0.12, -0.34, 0.56, ...]  # 注册时的特征（存在数据库）
feat_live       = [0.11, -0.35, 0.55, ...]  # 刚才拍到的特征

# 欧氏距离
distance = np.linalg.norm(
    np.array(feat_registered) - np.array(feat_live)
)

# 判断
if distance < 0.6:
    print("是同一个人，验证通过")
else:
    print("不是同一个人，验证失败")
```

**Dlib 的两个模型文件**（从网上下载，不需要训练）：
- `shape_predictor_68_face_landmarks.dat` — 检测脸上的 68 个关键点
- `dlib_face_recognition_resnet_model_v1.dat` — 提取 128 维特征向量

这两个文件是预训练好的，你的代码只负责"调用"。

## 4. 完整借用流程（所有模块串起来）

```
用户走到柜子前，说："你好储物柜，我要一个5V的稳压芯片"
│
├─ Step 1: 语音识别 (Person A)
│   Vosk 模型 → "你好储物柜，我要一个5V的稳压芯片"
│   VAD 检测到静音 → 语音结束
│
├─ Step 2: 唤醒词检测 (Person A)
│   "你好储物柜" → 触发交互状态机
│   实际查询文本 → "我要一个5V的稳压芯片"
│
├─ Step 3: LLM 语义匹配 (Person A)
│   ├─ Person A 调用 Person C 的 DataManager.get_all_components_for_llm()
│   │  → "1. LM7805 | 5V稳压 | 柜门1\n2. AMS1117 | 3.3V | 柜门2\n..."
│   ├─ 构造 prompt: 系统指令 + 器件目录 + few-shot + 用户查询
│   ├─ ctypes 调用 librkllmrt.so → NPU 推理
│   └─ 返回 → MatchResult(component_id=1, name="LM7805", cabinet_id=1, confidence=0.95)
│
├─ Step 4: 身份验证 (Person B → Person C 查数据库)
│   ├─ 人脸识别: 摄像头拍照 → Dlib 提取特征 → 跟数据库比对
│   │  或
│   └─ NFC 刷卡: PN532 读 UID → Person C 的 DataManager.get_user_by_card()
│      → UserRecord(id=2, name="张三", permission_level=1)
│
├─ Step 5: 开锁 (Person B)
│   hardware_controller.open_locker(cabinet_id=1)
│   → lgpio PWM → PWR.A153 B → SG90 舵机转 90°
│
├─ Step 6: 语音播报 (Person A)
│   eSpeak: "1号柜门已打开，请取用 LM7805"
│
└─ Step 7: 记录 (Person C)
    DataManager.create_borrow_record(user_id=2, component_id=1, cabinet_id=1)
    → SQLite INSERT INTO records ...
```

## 5. 三个人的接口关系

```
         ┌──────────────────────────────────────┐
         │           Person C（集成者）           │
         │                                      │
         │  SmartLockerSystem                    │
         │     │        │         │              │
         │     ▼        ▼         ▼              │
         │  DataManager  GUI    业务流程          │
         │     │                          │      │
         └─────┼──────────────────────────┼──────┘
               │                          │
    ┌──────────▼──────────┐    ┌─────────▼──────────┐
    │    Person A (AI)     │    │  Person B (硬件)    │
    │                      │    │                    │
    │  AIEngine            │    │  HardwareController│
    │  ├─ process_query()  │    │  ├─ open_locker()  │
    │  ├─ update_catalog() │    │  ├─ read_nfc_card()│
    │  └─ shutdown()       │    │  ├─ verify_face()  │
    │                      │    │  └─ shutdown()     │
    │  调用 Person C:       │    │                    │
    │  data_manager.        │    │  调用 Person C:     │
    │  get_all_components   │    │  data_manager.      │
    │  _for_llm()           │    │  get_user_by_card() │
    └──────────────────────┘    └────────────────────┘
```

**数据流方向**：
- Person A **读** Person C 的数据（器件目录）
- Person B **读** Person C 的数据（用户信息）
- Person C **调用** Person A 的方法（process_query）
- Person C **调用** Person B 的方法（open_locker）
